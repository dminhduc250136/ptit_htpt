**Bước 2: Fix disabled condition (D-13)**

Sau khi đọc, nếu bug là add-to-cart button luôn disabled (common pattern: `disabled={!product || product.stock === 0}` mà `product` chưa load), fix như sau:

Đảm bảo add-to-cart button chỉ disabled khi `product.stock === 0` (không phải khi `product.stock` là falsy 0 được cộng với condition khác):

```tsx
// Đúng pattern:
disabled={product.stock === 0}
// HOẶC nếu backend chưa trả stock (legacy fallback):
disabled={(product.stock ?? 0) === 0}
```

Quantity + button đúng pattern:
```tsx
disabled={quantity >= product.stock}
// KHÔNG dùng: disabled={quantity >= (product.stock ?? 99)} — ?? 99 làm cho button không bao giờ disabled khi stock thật < 99
// Đúng khi Plan 08-01 đã deploy (stock thật từ DB):
disabled={quantity >= (product.stock || 1)}
```

Quantity − button giữ nguyên: `disabled={quantity <= 1}`

**Bước 3: Thêm loading state cho add-to-cart button (D-03)**

Thêm state `addingToCart`:
```tsx
const [addingToCart, setAddingToCart] = useState(false);
```

Update add-to-cart onClick:
```tsx
onClick={async () => {
  setAddingToCart(true);
  try {
    addToCart(
      { id: product.id, name: product.name, thumbnailUrl: product.thumbnailUrl, price: product.price },
      quantity,
    );
    showToast('Đã thêm vào giỏ hàng', 'success');
  } catch {
    showToast('Không thể thêm vào giỏ hàng', 'error');
  } finally {
    setAddingToCart(false);
  }
}}
```

Update Button props:
```tsx
<Button
  size="lg"
  fullWidth
  disabled={product.stock === 0}
  loading={addingToCart}
  onClick={...}
>
  {product.stock === 0 ? 'Hết hàng' : 'Thêm vào giỏ hàng'}
</Button>
```
