<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->gateway | Public and admin request payloads arrive from untrusted clients. |
| gateway->service | Prefix routing can expose wrong endpoint surface if route assumptions drift. |
| admin route->service logic | Privileged operations require stronger authorization and audit guarantees. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | S | Admin user and admin product routes | mitigate | Enforce admin auth policy on admin-prefixed controllers and reject non-admin principals. |
| T-02-02-02 | T | Update/delete operations in user/product services | mitigate | Validate DTO constraints and whitelist mutable fields; preserve soft-delete policy by default. |
| T-02-02-03 | I | Product/user list endpoints | mitigate | Filter out soft-deleted records from public list endpoints and restrict admin-only data fields. |
| T-02-02-04 | R | Admin block/unblock and product status changes | mitigate | Record actor and trace-id context for state-changing operations for auditability. |
</threat_model>
