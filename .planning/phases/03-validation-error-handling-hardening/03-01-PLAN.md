## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> service API | Untrusted request body and query inputs enter validation pipeline |
| service exception mapper -> client response | Internal exception details must not leak via mapped error payload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | I | `GlobalExceptionHandler` rejectedValue output | mitigate | Mask sensitive field names and truncate oversized rejected values before serialization |
| T-03-02 | T | Error code mapping consistency | mitigate | Force common code taxonomy in all service handlers and protect with regression tests |
| T-03-03 | R | Traceability of failures | mitigate | Preserve `traceId` in all mapped error responses |
</threat_model>
