## Spacing Scale

Declared values — tất cả lấy từ CSS custom properties đã tồn tại:

| Token | CSS Var | Value | Usage |
|-------|---------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, badge padding, inline gap |
| sm | `--space-2` | 8px | Compact element spacing, button icon gap |
| md | `--space-3` | 16px | Form field gap, card inner padding |
| lg | `--space-4` | 24px | Section padding, modal padding |
| xl | `--space-5` | 32px | Card section padding (address form modal) |
| 2xl | `--space-6` | 48px | Page content padding, filter bar top spacing |
| 3xl | `--space-7` | 64px | Page-level section breaks |

**Exceptions:**

- Touch target minimum 44px cho mobile: `min-height: 44px` áp dụng cho `<button>` set-default, delete address, filter dropdown trigger — dùng padding bù, KHÔNG thêm token mới.
- AddressPicker dropdown trigger: `height: 40px` (Button size `md`) — consistent với checkout form buttons hiện tại.
- Filter bar inputs: `height: 40px` (native `<input type="date">` + keyword `<input>` + status `<select>`) — consistent với Button size `md`.

## Color

Tất cả token lấy từ `globals.css` — KHÔNG hardcode hex mới:

| Role | CSS Var | Hex (resolved) | Usage |
|------|---------|----------------|-------|
| Dominant (60%) | `--surface` / `--background` | #f7f9fb | Page background, filter bar background |
| Secondary (30%) | `--surface-container-lowest` | #ffffff | Address cards, modal dialog, order cards |
| Secondary alt | `--surface-container-low` | #f2f4f6 | Address list container background, filter bar container |
| Accent (10%) | `--primary` / `--gradient-primary` | #0056D2 | Xem danh sách dưới |
| Destructive | `--error` | #ba1a1a | Nút xóa địa chỉ (Button variant="danger") |

**Accent reserved for — danh sách explicit:**
1. Button "Thêm địa chỉ mới" (primary CTA trên `/profile/addresses`)
2. Button "Lưu địa chỉ" trong modal create/edit
3. Badge "Mặc định" — dùng `--primary-fixed` (#dae2ff) background + `--primary` text
4. AddressPicker dropdown trigger button (variant="secondary" với primary border tint)
5. Link "Xem chi tiết" trên order card (tertiary, underline on hover)
6. Filter "Áp dụng" button (nếu có explicit apply — xem section Filter Bar Interaction)
7. Active nav item trong profile sidebar (`--primary-fixed` bg + `--primary` text)

**Không dùng accent cho:** placeholder text, metadata text, card borders, dividers, input borders.

**Destructive pattern:**
- Nút "Xóa địa chỉ" → `Button variant="danger"` (background `--error` #ba1a1a)
- Confirmation dialog dùng Modal component hiện có với `primaryAction.variant="danger"`

## Layout Contracts

### `/profile/addresses` page

```
┌─────────────────────────────────────────────────────┐
│ Page Header (surface-container-low bg)              │
│   "Sổ địa chỉ"  [+ Thêm địa chỉ mới]               │
├─────────────────────────────────────────────────────┤
│ Layout: sidebar (260px) + main (1fr)                │
│ Sidebar: profile nav (reuse profile/page.tsx nav)   │
│ Main:                                               │
│   Address list — flex col, gap 12px                 │
│   ┌─ AddressCard ─────────────────────────────────┐ │
│   │ [Mặc định badge] fullName  phone              │ │
│   │ street, ward, district, city                  │ │
│   │ [Đặt mặc định] [Sửa] [Xóa]                   │ │
│   └───────────────────────────────────────────────┘ │
│   (empty state khi không có địa chỉ)               │

- Max addresses hiển thị: 10 (backend cap). Không cần pagination cho addresses.
- Khi đạt 10 addresses: button "Thêm địa chỉ mới" bị disabled + tooltip "Đã đạt giới hạn 10 địa chỉ".
- Mobile (< 768px): sidebar collapse, address list full width.

### AddressPicker trong `/checkout`

```
┌─ Checkout Form Column ────────────────────────────┐
│ [Địa chỉ đã lưu ▼]  ← Button secondary, full-    │
│                         width, đứng trên form      │
│ ┌─ Dropdown (khi open) ──────────────────────────┐│
│ │ ○ fullName — street, ward, district, city      ││
│ │ ● fullName — street, ward, district, city      ││
│ │   (default address pre-highlighted)            ││
│ └────────────────────────────────────────────────┘│
│ ─────── hoặc điền thủ công ──────────────────────│
│ [fullName input]    [phone input]                 │
│ [street input]                                    │
│ [ward input]    [district input]    [city input]  │
└───────────────────────────────────────────────────┘
```

- Dropdown width: 100% của form column.
- Dropdown max-height: 240px, overflow-y scroll.
- Default address: pre-highlighted với `--primary-fixed` background.
- Empty state (0 addresses): message "Chưa có địa chỉ đã lưu. [Thêm tại Sổ địa chỉ →]" — link mở `/profile/addresses` trong new tab.

```
┌─────────────────────────────────────────────────────┐
│ Page Header                                         │
│   "Lịch sử đơn hàng"                               │
├─────────────────────────────────────────────────────┤

- Filter bar: `display: flex`, `gap: --space-2` (8px), `flex-wrap: wrap` trên mobile.
- Filter bar background: `--surface-container-low`, `padding: --space-3 --space-4`, `border-radius: --radius-xl`.
- "Xóa bộ lọc" button: hiển thị khi có ít nhất 1 filter active (status != "ALL" hoặc from/to != empty hoặc q != empty). Button variant="tertiary".
- Filter apply: **live filtering** — mỗi khi user thay đổi bất kỳ input nào → debounce 400ms → encode URL → fetch. KHÔNG cần explicit "Áp dụng" button. Date inputs: apply khi blur hoặc change.

## AddressForm — Field Contract

Form dùng rhf + zod, áp dụng pattern Phase 10:

| Field | Label | Placeholder | Zod rule |
|-------|-------|-------------|----------|
| `fullName` | Họ và tên | "Nguyễn Văn A" | `z.string().min(2).max(100)` |
| `phone` | Số điện thoại | "0912345678" | `z.string().regex(/^(0|\+84)[3-9]\d{8}$/)` |
| `street` | Địa chỉ (số nhà, đường) | "123 Đường Lê Lợi" | `z.string().min(5).max(200)` |
| `ward` | Phường/Xã | "Phường Bến Nghé" | `z.string().min(2).max(100)` |
| `district` | Quận/Huyện | "Quận 1" | `z.string().min(2).max(100)` |
| `city` | Tỉnh/Thành phố | "TP. Hồ Chí Minh" | `z.string().min(2).max(100)` |

**Phone validation:** Dùng `^(0|\+84)[3-9]\d{8}$` — VN strict pattern, consistent với Phase 10 phone field.

**Form layout trong Modal:**
- 2 columns grid: fullName (1fr) + phone (1fr)
- street: full width (2 columns span)
- ward + district + city: 3 columns hoặc stack 1 column trên mobile

**Source:** CONTEXT.md D-04 (fields list) + Claude's Discretion (phone regex) + Phase 10 rhf/zod pattern

| State | Visual |

- Dùng `RetrySection` component hiện có (đã có)
- Checkout AddressPicker fetch fail: **silent** — ẩn picker button, KHÔNG show toast. User vẫn điền form thủ công bình thường. (Claude's Discretion — least disruptive option)

## Accessibility Contract

- Tất cả form fields: `<label>` element bắt buộc, `htmlFor` linked đúng id
- Modal: đã có `role="dialog"` + `aria-modal` + Esc close + scroll lock (Modal component hiện có)
- AddressPicker dropdown: `role="listbox"` + `aria-expanded` trên trigger button + `role="option"` cho từng item
- Filter status `<select>`: native element — không cần aria thêm
- Date inputs: native `<input type="date">` — không cần aria thêm
- "Xóa bộ lọc" button: `aria-label="Xóa tất cả bộ lọc đang áp dụng"`
- Badge "Mặc định": `aria-label="Địa chỉ mặc định"` hoặc sr-only text
- Disabled "Thêm địa chỉ mới" khi đạt limit: `aria-disabled="true"` + `title` tooltip
