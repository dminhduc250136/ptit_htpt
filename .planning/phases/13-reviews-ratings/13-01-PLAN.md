## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → user-service | Untrusted register/login input — fullName, username, password đến từ client |
| user-service → consumers (gateway, product-svc, order-svc) | JWT token signed HS256 — consumers verify signature trước khi tin claims |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01-01 | Tampering | JWT 'name' claim — attacker forge token với fullName tùy ý | mitigate | HS256 signature với jwtSecret server-side; consumers verify signature qua `Jwts.parser().verifyWith(...)` (đã có pattern trong JwtRoleGuard) — nếu signature fail → 401, không trust claim |
| T-13-01-02 | Spoofing | fullName chứa Unicode trick (homoglyph) hoặc HTML payload | accept (transfer downstream) | Phase 13 product-svc snapshot reviewerName vào DB rồi render plain text qua React text node + BE Jsoup sanitize content (D-14) — XSS surface đã được handle ở review render path. fullName tự nó chỉ hiển thị trong review UI (không exec); ASVS L1 không yêu cầu Unicode normalization ở v1.2 demo scale |
| T-13-01-03 | Information Disclosure | Token cũ thiếu claim 'name' → consumer crash NPE → leak stack trace | mitigate | Fallback ở 2 layer: (a) JwtUtils luôn set claim 'name' = fullName||username (Task 1); (b) ReviewController kiểm tra `claims.get("name")` null trước khi xài (sẽ implement ở Plan 03) |
| T-13-01-04 | Repudiation | User claim "tôi không phát review đó" — không có audit log | accept | Phase 13 demo scale; review entity lưu user_id từ JWT.sub đã đủ traceability cho v1.2. Audit log đầy đủ defer |
</threat_model>
