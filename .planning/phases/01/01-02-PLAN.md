# 01-02 — Swagger/OpenAPI enabled and consistent in every service
**Phase:** 1 — API Contract & Swagger Baseline  
**Plan item:** 01-02  
**Requirements:** API-01  

## Outcome (must be true)
- Every backend service exposes:
  - OpenAPI JSON at **`/v3/api-docs`**
  - Swagger UI reachable via **`/swagger-ui.html`** (redirect to `/swagger-ui/index.html` acceptable)
- Docs reflect real controllers/DTOs (at least smoke endpoints + any existing controllers).

## Work items (per service)

### 1) Add springdoc dependency (Spring MVC services)
- **Services**: `user-service`, `product-service`, `order-service`, `payment-service`, `inventory-service`, `notification-service`
- **Files**:
  - `sources/backend/<service>/pom.xml`
- **Action**: Add `org.springdoc:springdoc-openapi-starter-webmvc-ui` (version compatible with Spring Boot 3.3.x; keep consistent across services).
- **Verify**:
  - `cd sources/backend/<service>; mvn -DskipTests package`
- **Done when**: all 6 services build with springdoc.

### 2) Standardize springdoc paths
- **Files**:
  - `sources/backend/<service>/src/main/resources/application.yml`
- **Action**: Ensure (or confirm defaults):
  - `springdoc.api-docs.path=/v3/api-docs`
  - `springdoc.swagger-ui.path=/swagger-ui.html`
- **Verify**:
  - `curl -sS http://localhost:8080/v3/api-docs` (expect JSON)
  - `curl -I http://localhost:8080/swagger-ui.html`
- **Done when**: docs endpoints are reachable and actuator still works.

### 3) Ensure contract envelope does not break docs
- **Dependency**: requires 01-01 Task “ResponseBodyAdvice opt-outs”
- **Verify**:
  - `/v3/api-docs` JSON is not wrapped in the success envelope
  - Swagger UI loads assets correctly
- **Done when**: docs remain valid after response wrapping is introduced.

## Phase-level verification (roll-up)
- For each service, capture:
  - `GET /v3/api-docs` returns JSON
  - `GET /swagger-ui.html` returns 200/302
- If something is missing in the schema, add minimal annotations only where needed (prefer simple baseline).
