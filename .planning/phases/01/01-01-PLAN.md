# 01-01 — Standard API response + error schema
**Phase:** 1 — API Contract & Swagger Baseline  
**Plan item:** 01-01  
**Requirements:** API-03, API-04  

## Outcome (must be true)
- **All services** return a consistent **error schema** for 4xx/5xx.
- **At least one endpoint per service** returns a consistent **success envelope**.
- Status code conventions for CRUD are documented and verifiable (baseline conventions, not full CRUD completion).

## Contract (baseline)

### Success envelope

```json
{
  "timestamp": "2026-04-22T12:34:56.789Z",
  "status": 200,
  "message": "OK",
  "data": {}
}
```

### Error schema

```json
{
  "timestamp": "2026-04-22T12:34:56.789Z",
  "status": 400,
  "error": "Bad Request",
  "message": "Validation failed",
  "code": "VALIDATION_ERROR",
  "path": "/api/products/123",
  "traceId": "2f3b0b9d6b0f4b7a",
  "fieldErrors": [
    { "field": "name", "rejectedValue": "", "message": "must not be blank" }
  ]
}
```

### Error code baseline
- `VALIDATION_ERROR` → 400
- `BAD_REQUEST` → 400
- `UNAUTHORIZED` → 401
- `FORBIDDEN` → 403
- `NOT_FOUND` → 404
- `CONFLICT` → 409
- `INTERNAL_ERROR` → 500
- `SERVICE_UNAVAILABLE` → 503

## Work items (repeatable per service)

### 1) Add contract DTOs (per service)
- **Files (example `user-service`)**:
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/api/ApiResponse.java`
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/api/ApiErrorResponse.java`
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/api/FieldErrorItem.java`
- **Action**: Implement DTOs matching schemas above.
- **Verify**: `mvn -DskipTests package` compiles.
- **Done when**: DTOs exist in each service package and compile.

### 2) Add global exception handling (per service)
- **Files (example `user-service`)**:
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/api/GlobalExceptionHandler.java`
- **Action**:
  - Map JSON parse errors → `BAD_REQUEST`
  - Map validation errors → `VALIDATION_ERROR` with non-empty `fieldErrors`
  - Fallback exceptions → `INTERNAL_ERROR`
- **Verify** (after adding smoke endpoints below):
  - `curl -i -H "Content-Type: application/json" -d "{bad" http://localhost:8080/__contract/validate` returns 400 with `code=BAD_REQUEST` (or equivalent safe mapping)
- **Done when**: every service returns the standard error schema for at least parse errors + fallback.

### 3) Trace ID propagation in services
- **Files (example `user-service`)**:
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/web/TraceIdFilter.java`
- **Action**: Read `X-Request-Id` and make it available to the exception handler so it can populate `traceId`.
- **Verify**:
  - `curl -i -H "X-Request-Id: test-trace-1" -H "Content-Type: application/json" -d "{}" http://localhost:8080/__contract/validate`
  - Response includes `traceId: "test-trace-1"`
- **Done when**: `traceId` is present and stable per request.

### 4) Add minimal “contract smoke” endpoints (per service)
- **Files (example `user-service`)**:
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/web/ContractSmokeController.java`
- **Action**:
  - `GET /__contract/ping` returns `{ "service": "<service-name>" }`
  - `POST /__contract/validate` accepts a `@Valid` DTO (e.g., `name @NotBlank`) to force validation errors
- **Verify**:
  - `curl -i http://localhost:8080/__contract/ping` → 200
  - `curl -i -H "Content-Type: application/json" -d "{}" http://localhost:8080/__contract/validate` → 400 with `code=VALIDATION_ERROR` and non-empty `fieldErrors[]`
- **Done when**: each service has these two endpoints and they behave as specified.

### 5) Add success envelope wrapping (per service)
- **Files (example `user-service`)**:
  - `sources/backend/user-service/src/main/java/com/ptit/htpt/userservice/api/ApiResponseAdvice.java`
- **Action**: Wrap successful JSON responses in `ApiResponse<T>` using `ResponseBodyAdvice`.
- **Must NOT wrap**:
  - `/actuator/**`
  - `/v3/api-docs/**`
  - `/swagger-ui/**`
  - `/swagger-ui.html`
  - `/swagger-resources/**` (if present)
- **Verify**:
  - `curl -sS http://localhost:8080/__contract/ping` returns the success envelope with `data.service`
- **Done when**: smoke ping is consistently wrapped and springdoc/actuator endpoints remain unmodified.

## Phase-level verification (roll-up)
- Run each service and validate:
  - Error schema: `__contract/validate` invalid body / invalid JSON
  - Envelope schema: `__contract/ping`
