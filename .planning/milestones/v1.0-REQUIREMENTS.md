- [x] **API-01** (validated): Every service exposes Swagger/OpenAPI docs in a consistent way (including auth where applicable).
  - Springdoc `/v3/api-docs` + `/swagger-ui.html` enabled trên 6 services.
- [x] **API-02** (validated): API Gateway publishes an aggregated or discoverable OpenAPI surface for frontend integration.
  - Gateway swagger-ui aggregates 6 service specs; FE codegen consume thành 6 typed `.generated.ts` modules.
- [x] **API-03** (validated): Standardize response envelope and error format across services (error code, message, field errors, trace id).

- [x] **API-04** (validated): Standardize HTTP status code usage across services (success, validation, auth, not-found, conflict).
  - Common-code taxonomy enforced via `mapCommonCode` (VALIDATION_ERROR/UNAUTHORIZED/FORBIDDEN/NOT_FOUND/CONFLICT/INTERNAL_ERROR).

- [x] **VAL-01** (validated): Input validation is enforced consistently (request DTO validation, field-level messages).

- [x] **VAL-02** (validated): Global exception handling is consistent and maps to the standardized error format.
  - `GlobalExceptionHandler` triển khai 6/6 services với common-code taxonomy nhất quán.
- [x] **VAL-03** (validated): Auth/authz errors are consistent across gateway and services.
  - Gateway 6 regression tests PASS; caught dead-branch bug `NotFoundException 503→404`. `mapCommonCode` emits distinct UNAUTHORIZED/FORBIDDEN cả 2 phía.

### Frontend ↔ Backend Alignment

- [x] **FE-01** (validated): Frontend API client aligns to the documented contracts (URLs, DTOs, status codes, error format).
  - **Compile-time:** openapi-typescript codegen + httpGet/Post wrappers + 6 generated `.generated.ts` files compile clean.
  - **Runtime:** 3 ban đầu surfaced gaps đều closed via 04-04/05/06: (1) rich Product DTO; (2) CreateOrderCommand + X-User-Id + status=PENDING + totalAmount server-computed; (3) `/products/slug/{slug}` 200/404 (was 500). FE hardening WR-01/02/04 applied. Playwright A1/A3/A4/A5 all PASS với real backend.
- [x] **FE-02** (validated): Checkout and cart flows handle error cases gracefully (validation, stock, payment failure, auth).
  - 5 dispatcher branches verified per UAT B1..B5: B1 real-backend VALIDATION_ERROR → Banner + inline shippingAddress.* errors; B2 stubbed CONFLICT/STOCK_SHORTAGE → Stock modal; B3 stubbed CONFLICT/PAYMENT → Payment modal; B4a stubbed 401 → silent redirect + clearTokens (T-04-04); B5 stubbed 502 → Toast + no auto-retry (D-10). Open-redirect guard verified end-to-end (B4b real, T-04-03).
