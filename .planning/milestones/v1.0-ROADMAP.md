# Milestone v1.0 — MVP Stabilization (SHIPPED)

## Goal

Make the API surface consistent and complete (CRUD + Swagger/OpenAPI) and align frontend-backend contracts so E2E validation is reliable.

## Outcome

✅ **All 11 requirements met.** Shopping flow validated end-to-end via Playwright (12/12 PASS) on the live docker-compose stack: browse → cart → checkout → mock payment → confirmation. Cross-phase wiring (error envelope, traceId, gateway routing, codegen contract) verified WIRED in `v1.0-MILESTONE-AUDIT.md`.

## Key Accomplishments

1. **Standardized API contract** — 6 services + gateway emit unified `ApiErrorResponse` envelope (`{code, message, status, traceId, fieldErrors, path}`); traceId propagation via `X-Request-Id` header round-trip.
2. **Swagger/OpenAPI baseline** — Springdoc enabled on 6 services + gateway aggregator UI; frontend `npm run gen:api` produces 6 typed `.generated.ts` modules that compile clean.
3. **CRUD completeness across 6 services** — User/Product/Order/Payment/Inventory/Notification expose chuẩn REST (`GET list/by-id`, `POST`, `PUT`, `DELETE`) với pagination metadata, soft-delete baseline, và admin/public route boundaries.
4. **Validation & error handling hardened** — `@Valid` + `GlobalExceptionHandler` trong 6 services; gateway `GlobalGatewayErrorHandler` pass-through compliant downstream body + normalize incompatible. Caught and fixed dead-branch bug (`NotFoundException` 503→404).
5. **Frontend contract alignment** — Typed HTTP tier (`httpGet/Post/Put/Patch/Delete`) + `ApiError` dispatcher với 5 failure branches (VALIDATION_ERROR, CONFLICT/STOCK_SHORTAGE, CONFLICT/PAYMENT, 401, INTERNAL_ERROR); rich Product DTO + `CreateOrderCommand` wired E2E.
6. **E2E shopping flow validated** — Playwright UAT 12/12 PASS qua docker-compose stack thật; 5 failure cases verified (B1 real, B2/B3/B4a/B5 stubbed-with-deferred-evidence, B4b real returnTo guard).
