<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->gateway | Untrusted request payloads and query params cross into CRUD endpoints. |
| gateway->service | Routed prefixes can mismatch backend resource paths if not validated. |
| controller->repository | Unsafe update/delete logic can corrupt soft-delete state and data visibility. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | T | Inventory and notification create/update endpoints | mitigate | Validate request DTOs, enforce field bounds, and reject invalid enum/status transitions at controller entry. |
| T-02-01-02 | E | Admin-prefixed endpoints | mitigate | Enforce explicit admin route prefix separation and admin auth policy checks for admin controllers only. |
| T-02-01-03 | R | Soft-delete operations | mitigate | Persist actor/time metadata for delete/update operations and keep trace id propagation in responses. |
| T-02-01-04 | D | Gateway route fan-out | accept | Prefix wildcard routing is existing architecture; mitigate operational risk via same-cycle smoke script and health checks. |
</threat_model>
