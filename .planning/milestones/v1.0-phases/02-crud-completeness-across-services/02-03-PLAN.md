## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->gateway | Checkout/cart/payment inputs are untrusted and may be malformed or replayed. |
| gateway->order/payment services | Prefix routing errors can misdirect lifecycle operations. |
| payment callback/public endpoint->payment logic | Public callback routes risk tampering and replay. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | T | Order and payment mutation endpoints | mitigate | Validate request DTO schema and enforce state transition guards in service layer. |
| T-02-03-02 | S | Admin order endpoints | mitigate | Restrict `/admin/orders/**` to admin principals and maintain route separation from public APIs. |
| T-02-03-03 | R | Order/payment state changes | mitigate | Include trace-id and actor context in state transition logs and error responses. |
| T-02-03-04 | D | Payment public callback routes | mitigate | Enforce idempotent callback handling and reject invalid signatures/duplicate transitions in payment service logic. |
</threat_model>
