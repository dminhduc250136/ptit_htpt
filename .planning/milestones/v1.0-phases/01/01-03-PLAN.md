# 01-03 — Gateway API surface conventions + compatibility checks
**Phase:** 1 — API Contract & Swagger Baseline  
**Plan item:** 01-03  
**Requirements:** API-02, API-03  

## Outcome (must be true)
- Gateway provides a **single entry point** for docs and request correlation:
  - Swagger UI reachable at `http://localhost:8080/swagger-ui.html`
  - UI lists all services via gateway-proxied `/v3/api-docs` endpoints
- Gateway ensures `X-Request-Id` exists and is forwarded to services.
- Gateway returns the **standard error schema** for gateway-local errors (route miss, etc.) without rewriting downstream service errors.

## Work items (gateway)

### 1) Add springdoc to gateway (WebFlux)
- **Files**:
  - `sources/backend/api-gateway/pom.xml`
- **Action**: Add `org.springdoc:springdoc-openapi-starter-webflux-ui` (compatible with Spring Boot 3.3.x).
- **Verify**:
  - `cd sources/backend/api-gateway; mvn -DskipTests package`
- **Done when**: gateway builds with springdoc.

### 2) Configure gateway-hosted Swagger UI for multi-service docs (API-02)
- **Files**:
  - `sources/backend/api-gateway/src/main/resources/application.yml`
- **Action**: Add/merge:

```yml
springdoc:
  swagger-ui:
    path: /swagger-ui.html
    urls:
      - name: users
        url: /api/users/v3/api-docs
      - name: products
        url: /api/products/v3/api-docs
      - name: orders
        url: /api/orders/v3/api-docs
      - name: payments
        url: /api/payments/v3/api-docs
      - name: inventory
        url: /api/inventory/v3/api-docs
      - name: notifications
        url: /api/notifications/v3/api-docs
```

- **Verify** (services must be running):
  - `curl -I http://localhost:8080/swagger-ui.html`
  - `curl -sS http://localhost:8080/api/users/v3/api-docs` (expect JSON)
  - Repeat for all services in the list
- **Done when**: gateway Swagger UI lists and loads each service spec.

### 3) Ensure gateway routes actually proxy `/v3/api-docs`
- **Files**:
  - `sources/backend/api-gateway/src/main/resources/application.yml`
- **Action**: If current rewrite rules block docs paths, adjust rewrite filters so `/api/<prefix>/v3/api-docs` reaches downstream `/v3/api-docs`.
- **Verify**:
  - `curl -sS http://localhost:8080/api/products/v3/api-docs` returns OpenAPI JSON from product-service
- **Done when**: docs proxying works for every service.

### 4) Request ID propagation (API-03)
- **Files**:
  - `sources/backend/api-gateway/src/main/java/com/ptit/htpt/apigateway/gateway/RequestIdFilter.java`
- **Action**:
  - If `X-Request-Id` present, keep it.
  - Else generate a short UUID and set it.
  - Ensure it is forwarded downstream.
- **Verify**:
  - Call a service endpoint through gateway with no `X-Request-Id`, then force an error and confirm `traceId` is populated.
- **Done when**: every request has a usable trace id.

### 5) Gateway-local error schema (API-03)
- **Files**:
  - `sources/backend/api-gateway/src/main/java/com/ptit/htpt/apigateway/gateway/GlobalGatewayErrorHandler.java`
- **Action**:
  - For gateway-local 404 / route miss, return the standard error schema with `code=NOT_FOUND` and include `traceId`.
  - Do not double-wrap or transform downstream service error bodies (pass-through).
- **Verify**:
  - `curl -i http://localhost:8080/api/does-not-exist/foo` returns standard error schema with `code=NOT_FOUND`
- **Done when**: gateway-local errors are consistent with service errors.
