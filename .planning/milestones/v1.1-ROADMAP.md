# Milestone v1.1 — Real End-User Experience (SHIPPED)

## Goal

Biến demo flow từ "stub-verified" thành "real visible end-to-end" — mọi thứ user click trên UI hoạt động với real data thay vì mock/seeded. Audit phát hiện v1.0 chạy in-memory (không có DB layer) → v1.1 mở đầu bằng cluster C0 Database Foundation block C1/C2/C3.

## Outcome

✅ **15/19 requirements SATISFIED + 4/19 PARTIAL** (xem audit). Flow shopping E2E hoạt động với real data: browse từ DB → cart → checkout → order detail full breakdown. Auth thật + admin CRUD + stock persistence — toàn bộ stack chạy trên Postgres + JPA + Flyway, không còn in-memory.

4 partial gaps (AUTH-06 middleware narrow, UI-02 admin landing dashboard, PERSIST-01 cart 409 handling, DB-05 V2 collision) đều có **compensating controls hoặc fixes commited cuối milestone**. Defer phần residual sang v1.2.

## Key Accomplishments

1. **Database Foundation thật** — Postgres 16 container vào docker-compose; 5 services (user/product/order/payment/inventory) refactor in-memory → JPA + Flyway baseline (V1) + dev seed (V2 từ FE mocks); `sources/frontend/src/mock-data/` đã xóa hoàn toàn; gateway round-trip `GET /api/products` trả seeded data từ DB.
2. **Auth flow thật** — Backend `/api/users/auth/{register,login,logout}` với BCrypt + JWT HS256 24h; FE login/register pages gỡ mock + auto-login; middleware admin role gate → /403; AuthProvider hydration cho session persist.
3. **Admin + Search real CRUD** — `/search` keyword filter qua gateway; `admin/products` create/edit/delete + V2 migration thêm brand/thumbnail/shortDescription/originalPrice; `admin/orders` list + detail page với status update; `admin/users` PATCH fullName/phone/roles + soft-delete (CUSTOMER only).
4. **Cart → Order persistence visible** — `ProductEntity.stock` persist (V3 migration); `OrderEntity` với per-item `OrderItemEntity` rows + JSON `shippingAddress` + `paymentMethod`; FE `/profile/orders/{id}` async fetch render full breakdown 4-column items table; checkout redirect tới `/profile/orders/{id}` sau thành công.
5. **Cross-cutting fixes** — Gateway two-route-per-service pattern (base + wildcard) tránh trailing-slash + `$PATH` env conflict; FE `services/products.ts` paths corrected; debug session "login redirect loop + cart stock bypass" resolved + V2 Flyway collision fix.
